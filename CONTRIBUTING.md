# Contributing

Thanks for your interest in My Hermes Desktop.

## Issues

- Use the bug report template for reproducible problems. Include system version, Hermes CLI version, screenshots, and logs when available.
- Use the feature request template for new ideas. Describe the use case and expected behavior.
- Search existing issues before opening a duplicate.

## Pull Requests

1. Fork the repository and create a feature branch.
2. Keep changes focused and reviewable.
3. Run type checks and builds before submitting.
4. Describe what changed, why it changed, and how it was verified.
