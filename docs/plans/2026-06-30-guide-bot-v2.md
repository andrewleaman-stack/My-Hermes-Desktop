# Guide Bot v2 Implementation Plan

> **For Hermes:** Use subagent-driven-development skill if this expands beyond one focused patch set.

**Goal:** Turn the composer Guide Bot into a tiny live operations console for Hermes runs, while skipping onboarding entirely.

**Architecture:** Keep the UI in `GuideBot.tsx`, driven by the existing `GuideActivity` stream state from `ChatPage.tsx`. Add client-side latency metrics, auto density, an inspector popover, personality levels, richer tool labels, and more expressive faces without adding a new backend service.

**Tech Stack:** React + TypeScript, Tauri event stream, CSS animations/localStorage.

---

## Out of Scope

- No onboarding/tutorial nudges.
- No new agent/task system.
- No persistent cloud telemetry; all latency stats stay in localStorage.

## Task 1: Preferences

**Objective:** Add `auto` display mode and personality levels.

**Files:**
- Modify: `src/components/chat/GuideBot.tsx`
- Modify: `src/pages/SettingsPage.tsx`
- Modify: `src/index.css`

**Details:**
- Extend `GuideBotDisplayMode` to `"auto" | "full" | "compact" | "hidden"`.
- Add `GuideBotPersonality = "minimal" | "helpful" | "playful"`.
- Store in `localStorage` using keys `hermes.guideBot.displayMode` and `hermes.guideBot.personality`.
- Settings should expose both controls.

## Task 2: Latency Metrics

**Objective:** Track local latency and show slow-provider hints.

**Files:**
- Modify: `src/types.ts`
- Modify: `src/pages/ChatPage.tsx`
- Modify: `src/components/chat/GuideBot.tsx`

**Details:**
- Extend `GuideActivity` with `firstTokenAt?: number`, `completedAt?: number`, and `lastTool?: string`.
- Set `firstTokenAt` on first `text` chunk.
- Set `completedAt` on `done`.
- Store rolling local averages in `localStorage` from completed runs.
- Guide Bot copy should distinguish no-output latency from normal streaming.

## Task 3: Auto Density

**Objective:** Make the bot full only when useful.

**Files:**
- Modify: `src/components/chat/GuideBot.tsx`
- Modify: `src/index.css`

**Rules:**
- Full when error, alert, queued, slow latency, or inspector is open.
- Compact for normal streaming/typing.
- Minimal idle footprint.
- Hidden still hides permanently.

## Task 4: Inspector Popover

**Objective:** Clicking the avatar opens a small run inspector instead of only focusing input.

**Files:**
- Modify: `src/components/chat/GuideBot.tsx`
- Modify: `src/index.css`

**Inspector shows:**
- phase
- elapsed
- time to first token if available
- model
- queue length
- context usage
- last tool/detail
- actions: focus, stop, clear queue, retry, compress when relevant

## Task 5: Tool Labels + Faces

**Objective:** Make tool states and expressions more legible.

**Files:**
- Modify: `src/components/chat/GuideBot.tsx`
- Modify: `src/index.css`

**Details:**
- Humanize known tool details: `read_file`, `terminal`, `patch`, `search_files`, `write_file`, `image_generate`, etc.
- Add stopped/waiting/tool-ish face variants using CSS classes, without new assets.

## Task 6: Verification

Run:
- `npm run build`
- `cargo check` from `src-tauri`
- `npm test`
- `npm run tauri build`

Install/relaunch:
- Replace `/Applications/My Hermes Desktop.app`
- Open it and verify process is running.
